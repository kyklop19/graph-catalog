graph_catalog.catalog.load_with_number(number: str) → dict[str, Any][zdroj]

Loads graph and metadata from catalog using graph’s number

To make loading with number anonymous, the function doesn’t deletes anonymous fields from the metadata before returning.

Parametry:

number (str) – Number of the graph in format „<number>.<number>“

Vyvolá:

  • KeyError – Raises when number is in a wrong format

  • KeyError – Raises when graph with such number doesn’t exist

Vrací:

Dictionary with the graph in IncMat representation and it’s metadata except the anonymous fields

graph_catalog.catalog.save(name: str, filename: str, graph: IncMat, metadata: dict[str, Any] = {}, opts: SaveOpts | None = None) → str[zdroj]

Save graph as IncMat to the catalog

The graph is saved under the name of name into the file filename.yaml.

metadata are saved with graph into filename.yaml

number of the graph is generated and returned.

By default graph can’t be overwritten resp. can’t be written into nonexisting file. By setting opts to SaveOpts.OVERWRITE_GRAPH resp. SaveOpts.CREATE_NEW_FILE this behavior can be turned off.

Parametry:

  • name (str) – Name of the graph

  • filename (str) – Name of the file where graph is saved (without the extension)

  • graph (IncMat) – Graph as IncMat

  • metadata (dict[str, Any], optional) – Dictionary with field names as keys and any data as values. Defaults to {}.

  • opts (SaveOpts, optional) – Options to either allow overwriting a graph or creating new graph file. Defaults to None.

Vyvolá:

  • FileNotFoundError – Raises if filename.yaml doesn’t exists unless SaveOpts.CREATE_NEW_FILE is set as option

  • OverwriteError – Raises if name graph already exists unless SaveOpts.OVERWRITE_GRAPH is set as option

Vrací:

The number of the graph automatically generated based of filename.yaml and number of already saved files in it

Typ návratové hodnoty:

str
